# LOL 4.20 AI Bot 行为逻辑设计文档

## 一、概述

本文档详细描述《英雄联盟》4.20版本中AI Bot的行为逻辑设计。基于Riot Games官方开发博客和游戏实际运行数据，采用**分层行为树(Layered Behavior Tree)**架构，实现智能、多样且符合游戏策略的Bot行为。

**核心设计理念**：
- **类人化行为**：Bot行为应尽可能模拟人类玩家，包括决策方式、操作习惯和策略选择
- **动态威胁评估**：基于血量、技能状态、敌方人数等多维度评估战斗风险
- **角色差异化**：不同定位的英雄表现出截然不同的行为模式
- **团队协作**：Bot之间能够进行有效的通信和配合

---

## 二、架构设计

### 2.1 行为树架构

```
Root (Selector)
├── 紧急行为层 (Emergency Layer) - 最高优先级 [Priority: 100]
│   ├── 死亡检测
│   ├── 防御塔躲避
│   ├── 致命伤害规避
│   └── 生命值紧急恢复
│
├── 战斗行为层 (Combat Layer) [Priority: 80]
│   ├── 目标检测与评估
│   ├── 攻击范围判断
│   ├── 技能释放系统
│   ├── 普攻控制与走A
│   ├── 追击/逃跑决策
│   └── 威胁评估系统
│
├── 对线行为层 (Laning Layer) [Priority: 60]
│   ├── 兵线管理与状态判断
│   ├── 补刀控制
│   ├── 推线策略
│   ├── 换血决策
│   └── 视野控制
│
├── 资源管理层 (Resource Layer) [Priority: 40]
│   ├── 回城决策
│   ├── 装备购买系统
│   ├── 技能冷却管理
│   ├── 血量/蓝量恢复
│   └── 召唤师技能管理
│
└── 宏观行为层 (Macro Layer) [Priority: 20]
    ├── 路线规划
    ├── 支援决策
    ├── 野区控制
    ├── 目标推进
    └── 团队协作
```

### 2.2 行为树节点类型

| 节点类型 | 说明 | 示例 |
|---------|------|------|
| **Selector** | 选择器，依次执行子节点，返回第一个成功的 | 战斗行为选择 |
| **Sequence** | 序列器，依次执行子节点，全部成功才返回成功 | 回城流程 |
| **Decorator** | 装饰器，修改子节点行为 | 条件判断、循环、概率控制 |
| **Leaf** | 叶子节点，执行具体动作 | 移动、攻击、释放技能 |
| **Parallel** | 并行执行多个子节点 | 同时监控多个威胁 |

## 三、核心行为模块

### 3.1 紧急行为层

#### 3.1.1 死亡检测
- **触发条件**: 血量 <= 0
- **行为**: 停止所有动作，等待重生
- **重生后行为**: 根据当前游戏时间和角色定位决定行动目标

#### 3.1.2 防御塔躲避

**触发条件**:
- 进入敌方防御塔攻击范围（约800单位）
- 未在战斗中或战斗持续时间超过5秒

**躲避策略**:
```
如果 进入防御塔范围:
    如果 正在攻击敌人且敌人血量 < 20%:
        继续攻击直到击杀或自身血量 < 20%
    否则:
        向防御塔攻击范围外移动
        移动方向 = 远离防御塔的最近方向
        移动速度 = 最大移动速度 * 1.2
```

**优先级**: 高于战斗行为（除非正在进行关键击杀）

#### 3.1.3 致命伤害规避

**检测范围**: 1500单位

**规避对象**: 
- 指向性技能（技能弹道）
- 范围技能（技能指示器）
- 敌方英雄突进技能

**规避策略**:
```
当检测到技能威胁时:
    计算技能到达时间 = 技能距离 / 技能速度
    如果 技能到达时间 < 反应时间(0.5-1.0秒):
        向技能方向的垂直方向移动
        移动距离 = 最大移动速度 * 反应时间
    否则:
        保持当前行为，但准备规避
```

**技能躲避概率**:
| 难度 | 躲避概率 | 反应时间 |
|------|---------|---------|
| 简单 | 20% | 1.0秒 |
| 普通 | 40% | 0.8秒 |
| 困难 | 60% | 0.6秒 |
| 地狱 | 80% | 0.4秒 |

**方向感知**: Bot会根据敌人面向方向预判技能释放方向

#### 3.1.4 生命值紧急恢复

**触发条件**: 血量 < 15%

**恢复策略**:
- 如果有治疗药水，立即使用
- 如果有治疗技能（如索拉卡W、奶妈技能），立即释放
- 如果在防御塔范围内，保持不动
- 否则，向最近的安全区域移动

### 3.2 战斗行为层

#### 3.2.1 目标检测与评估

**检测优先级算法**:
```
目标得分 = 基础优先级 * 权重系数

基础优先级:
1. 正在攻击自己的敌人: +500分
2. 生命值最低的敌人: +(1 - 血量百分比) * 300
3. 距离最近的敌人: +max(0, 200 - 距离/10)
4. 敌方ADC/APC: +200分
5. 正在攻击己方队友的敌人: +150分
6. 近战英雄: +50分（更容易击杀）
7. 被控制的敌人: +100分

权重系数:
- 攻击范围内: 1.0
- 技能范围内: 0.8
- 超出技能范围: 0.5
```

**检测范围**:
- 近战英雄: 600-1200单位
- 远程英雄: 800-1800单位

**目标切换条件**:
- 当前目标死亡
- 当前目标超出攻击范围超过2秒
- 出现更高优先级的目标

#### 3.2.2 威胁评估系统

**威胁评估算法**（基于Riot 2014年开发博客）:

```
威胁值 = 自身威胁 + 敌方威胁

自身威胁:
- 血量百分比: +血量% * 100
- 可用技能数量: +可用技能数 * 50
- 装备评分: +装备总价值 / 100
- 召唤师技能可用: +每个可用技能 * 30

敌方威胁:
- 敌方人数: +敌方人数 * 80
- 敌方血量百分比: +平均血量% * 50
- 敌方可用技能数量: +敌方可用技能总数 * 30
- 距离防御塔: -距离/10（越近威胁越低）

战斗决策:
如果 自身威胁 > 敌方威胁 * 1.2:
    进攻
如果 自身威胁 > 敌方威胁 * 0.8:
    保持战斗
否则:
    撤退
```

**关键设计理念**（来自Riot官方）:
- 如果Veigar释放完所有技能但Bot存活，Bot应该反击，因为Veigar在技能冷却期间几乎没有伤害潜力
- Bot不仅考虑血量，还考虑技能状态和装备

#### 3.2.3 攻击范围判断

```
距离计算 = 欧几里得距离(自身位置, 目标位置)
攻击范围 = 基础攻击距离 + 技能加成 + 装备加成

如果 距离 <= 攻击范围 + 50:
    执行普攻
    如果 攻击后摇时间 > 0:
        走A: 向最优位置移动
否则:
    移动至攻击范围
    移动路径 = 最短路径(自身位置, 目标位置 - 攻击范围偏移)
```

#### 3.2.4 技能释放系统

**技能释放优先级**:
```
1. 控制技能: 先手控制、打断技能 [权重: 100]
   - 目标: 敌方ADC/APC
   - 条件: 目标在范围内，队友在附近

2. 伤害技能: 高伤害技能、连招技能 [权重: 80]
   - 目标: 当前攻击目标
   - 条件: 蓝量足够，技能冷却完成

3. 位移技能: 追击或逃跑 [权重: 60]
   - 追击时: 向目标方向位移
   - 逃跑时: 向安全方向位移

4. 被动技能: 自动触发 [权重: 40]

5. 治疗/护盾技能: 保护自己或队友 [权重: 70]
   - 目标: 血量最低的己方单位
   - 条件: 目标血量 < 50%
```

**连招系统**:
- 每个英雄有预设连招组合（如安妮: Q-W-E-R）
- 根据技能冷却和蓝量选择最优连招
- 考虑技能施法时间和弹道速度
- 连招成功率: 简单40% -> 普通60% -> 困难80% -> 地狱95%

**技能释放条件**:
- 蓝量足够（至少保留10%蓝量应急）
- 技能冷却完成
- 目标在技能范围内（考虑技能弹道）
- 不会误伤到队友（AOE技能需要检查范围内队友）
- 目标不是免疫该技能状态

**技能命中率**:
| 难度 | 指向技能 | 非指向技能 | AOE技能 |
|------|---------|-----------|--------|
| 简单 | 50% | 30% | 40% |
| 普通 | 70% | 50% | 60% |
| 困难 | 85% | 70% | 80% |
| 地狱 | 95% | 85% | 90% |

#### 3.2.5 普攻控制与走A

**走A机制**:
```
攻击间隔: 基础攻击间隔 / (1 + 攻击速度加成)
攻击前摇: 基础前摇时间
攻击后摇: 基础后摇时间

走A循环:
1. 开始攻击前摇
2. 攻击命中目标
3. 立即开始移动（取消后摇）
4. 移动至最优攻击位置
5. 等待攻击间隔结束
6. 重复步骤1
```

**最优攻击位置计算**:
- 远程英雄: 保持在最大攻击范围边缘
- 近战英雄: 保持在目标周围，避免被风筝
- 考虑敌方技能范围，保持安全距离

**攻击目标切换**:
- 优先攻击当前目标
- 目标死亡或超出范围时切换目标
- 考虑目标威胁程度

#### 3.2.6 追击/逃跑决策

**追击条件**:
```
追击评分 = 
    (目标血量百分比 * -100) +      # 血量越低越值得追击
    (自身血量百分比 * 50) +        # 自身血量越高越敢追击
    (距离目标距离 * -0.5) +        # 距离越近越容易追击
    (是否有位移技能 * 100) +        # 有位移更容易追击
    (队友数量 * 50) -               # 队友越多越安全
    (敌方队友数量 * 80)             # 敌方队友越多越危险

如果 追击评分 > 50:
    追击目标
    追击距离上限: 1500单位
    追击时间上限: 5秒
```

**逃跑条件**:
```
逃跑评分 = 
    (自身血量百分比 * -100) +       # 血量越低越需要逃跑
    (敌方人数 * 100) -              # 敌方人数越多越危险
    (队友人数 * 50) +               # 队友越多越安全
    (是否受到控制 * 200) +          # 被控制时必须逃跑
    (距离防御塔距离 * -0.3)         # 越靠近防御塔越安全

如果 逃跑评分 > 100:
    逃跑
    逃跑方向: 最近的防御塔或队友方向
    逃跑时使用位移技能
```

### 3.3 对线行为层

#### 3.3.1 兵线管理与状态判断

**三种兵线状态**:
| 状态 | 特征 | 策略 |
|------|------|------|
| **Freeze（冻结）** | 兵线靠近己方防御塔（距离<500） | 只补最后一刀，压制敌人经济 |
| **Slow Push（慢推）** | 己方兵线数量占优（>敌方+2） | 逐步推进，创造越塔机会 |
| **Fast Push（快推）** | 快速清线 | 快速推线后回城或支援 |

**兵线判断算法**:
```
兵线状态 = 计算兵线趋势()

计算兵线趋势():
    己方小兵数量 = count(己方小兵在对线区域)
    敌方小兵数量 = count(敌方小兵在对线区域)
    己方小兵平均血量 = avg(己方小兵血量)
    敌方小兵平均血量 = avg(敌方小兵血量)
    兵线位置 = 距离己方防御塔距离

    如果 己方小兵数量 > 敌方小兵数量 + 2:
        return FAST_PUSH
    elif 兵线位置 < 500:
        return FREEZE
    else:
        return SLOW_PUSH
```

**推线时机**:
- 敌方英雄回城
- 准备支援其他路线
- 己方打野准备gank
- 需要快速回城补给
- 己方防御塔即将被摧毁

#### 3.3.2 补刀控制

**补刀优先级**:
1. 炮兵（最高价值，190金币）
2. 近战小兵（前期60金币，随时间增长）
3. 远程小兵（前期30金币，随时间增长）

**补刀时机算法**:
```
如果 小兵血量 <= 自身攻击力 * 1.2:
    执行补刀
    补刀成功率: 简单70% -> 普通85% -> 困难95% -> 地狱99%
否则:
    等待小兵血量降低
    保持在攻击范围内
```

**补刀策略**:
- 预判小兵死亡时间
- 计算自身伤害是否足以击杀
- 考虑敌方英雄干扰
- 远程英雄优先使用技能补刀炮兵

#### 3.3.3 推线策略

**推线方法**:
- 使用技能快速清线（技能清线效率高时）
- 攻击所有小兵而非只补刀
- 必要时越塔推线（有队友配合时）

**推线后的行为**:
- 回城补给
- 支援其他路线
- 入侵敌方野区
- 控制小龙/大龙视野

#### 3.3.4 换血决策

**换血条件**:
```
换血评分 = 
    (自身血量百分比 - 敌方血量百分比) * 100 +
    (技能命中率估计 * 50) +
    (是否有打野威胁 * -100) +
    (距离防御塔距离 * 0.1)

如果 换血评分 > 30 且 自身血量 > 60%:
    执行换血
    换血方式: 使用技能消耗后立即退回安全位置
```

**换血策略**:
- 使用技能消耗后立即退回安全位置
- 避免过度深入敌方区域
- 计算换血性价比（伤害输出/受到伤害）
- 优先换血目标: 敌方血量较低的英雄

#### 3.3.5 视野控制

**插眼位置**:
- 河道入口（控制小龙视野）
- 三角草丛（防gank）
- 敌方野区入口（控制打野动向）
- 小龙/大龙坑附近（控制资源视野）

**视野更新**:
- 定期检查眼位是否过期（假眼3分钟，真眼1分钟）
- 根据游戏时间调整眼位策略
- 团战前确保关键区域有视野
- 辅助英雄优先插眼

### 3.4 资源管理层

#### 3.4.1 回城决策

**回城条件**:
```
回城评分 = 
    (1 - 血量百分比) * 100 +        # 血量越低越需要回城
    (1 - 蓝量百分比) * 50 +         # 蓝量越低越需要回城
    (是否有足够金币买装备 * 80) +   # 有金币买装备时回城
    (兵线位置评分) +                # 兵线安全时回城
    (是否有队友在线 * -30)          # 队友在线时可以不回城

兵线位置评分:
    如果 兵线在敌方防御塔附近: +50
    如果 兵线在己方防御塔附近: -30
    如果 兵线在中间: 0

如果 回城评分 > 60:
    准备回城
    回城时机: 敌方英雄不在线上
```

**回城时机**:
- 敌方英雄不在线上
- 兵线安全（不会被快速推塔）
- 没有即将发生的团战
- 击杀敌人后15秒内不回城（避免战斗中回城）

#### 3.4.2 装备购买系统

**装备优先级**:
```
1. 核心装备: 根据英雄定位和版本强势装备 [权重: 100]
   - ADC: 无尽之刃、饮血剑、幻影之舞
   - AP: 帽子、法穿鞋、沙漏
   - 坦克: 日炎、反甲、蓝盾
   - 辅助: 香炉、救赎、坩埚

2. 防御装备: 根据敌方阵容 [权重: 80]
   - 敌方AP多: 魔抗装备
   - 敌方AD多: 护甲装备
   - 敌方爆发高: 血量装备

3. 输出装备: 提升伤害能力 [权重: 60]

4. 鞋子: 根据需求选择（攻速、冷却、移速）[权重: 50]
```

**购买策略**:
- 回城时优先购买当前金币能买的最优装备
- 考虑装备合成路线（优先买小件）
- 根据敌方英雄调整装备选择
- 避免重复购买相同装备

**装备购买逻辑**:
```
for 每个可用装备:
    计算装备性价比 = 装备属性增益 / 装备价格
    
选择性价比最高且金币足够的装备
如果 没有足够金币买任何装备:
    购买小件或药水
```

#### 3.4.3 技能冷却管理

**技能状态追踪**:
- 记录每个技能的冷却时间
- 计算技能可用时间
- 在战斗中优先使用冷却完成的技能

**技能预读**:
- 预判技能冷却完成时间
- 在团战前规划技能使用顺序
- 根据技能冷却状态调整战斗策略

**技能使用策略**:
- 团战时: 先释放控制技能，再释放伤害技能
- 对单时: 根据连招顺序释放技能
- 逃跑时: 保留位移技能用于逃跑

#### 3.4.4 血量/蓝量恢复

**恢复策略优先级**:
1. 使用治疗药水（血量<50%时）
2. 使用技能恢复（如生命偷取、法术吸血）
3. 回到安全区域自然恢复
4. 利用防御塔回血效果
5. 等待队友治疗

**药水使用条件**:
- 血量 < 50%（红药水）
- 蓝量 < 30%（蓝药水）
- 没有正在受到伤害
- 不在战斗状态

#### 3.4.5 召唤师技能管理

**召唤师技能优先级**:
```
治疗:
    目标: 血量最低的己方单位（包括自己）
    条件: 目标血量 < 30% 或 有队友即将死亡

虚弱:
    目标: 正在攻击己方ADC的敌方英雄
    条件: 敌方英雄正在输出高伤害

引燃:
    目标: 正在逃跑的敌方英雄
    条件: 目标血量 < 20% 且 没有治疗效果

闪现:
    追击时: 向目标方向闪现
    逃跑时: 向安全方向闪现
    开团时: 向敌方ADC方向闪现

传送:
    支援时: 传送到友军小兵或眼位
    防守时: 传送到己方防御塔
    推塔时: 传送到前线

净化:
    目标: 自己
    条件: 受到控制效果且正在被集火

屏障:
    目标: 自己
    条件: 即将受到致命伤害
```

### 3.5 宏观行为层

#### 3.5.1 路线规划

**路线优先级**:
```
1. 线上: 保持对线压制 [权重: 100]
   - 条件: 兵线在中间或己方优势

2. 野区: 打野英雄清理野怪 [权重: 80]
   - 条件: 野怪刷新且线上队友安全

3. 支援: 帮助队友获得优势 [权重: 70]
   - 条件: 队友正在被攻击且支援路线安全

4. 推进: 推塔或拿龙 [权重: 60]
   - 条件: 己方人数优势且兵线良好
```

**路线选择算法**:
```
路线评分 = 
    (路线收益 * 权重) + 
    (路线风险 * -权重) + 
    (当前状态加成)

路线收益:
    线上: 补刀收益 + 经验收益
    野区: 野怪金币 + BUFF收益
    支援: 击杀概率 * 击杀奖励
    推进: 防御塔金币 + 地图控制权

路线风险:
    线上: 被gank概率 * 死亡损失
    野区: 被反野概率 * 死亡损失
    支援: 支援失败概率 * 时间损失
    推进: 被防守概率 * 死亡损失

选择评分最高的路线
```

#### 3.5.2 支援决策

**支援条件**:
```
支援评分 = 
    (队友血量百分比 * -100) +      # 队友血量越低越需要支援
    (队友与敌人数量差 * 50) +      # 人数差距越大越需要支援
    (支援距离 * -0.5) +            # 距离越近越容易支援
    (自身血量百分比 * 30) +        # 自身血量越高越能支援
    (是否有位移技能 * 50)          # 有位移更容易支援

如果 支援评分 > 50:
    执行支援
```

**支援策略**:
- 优先支援小龙/大龙附近的战斗
- 支援时使用最快路径
- 考虑是否需要绕后或正面进场
- 支援后评估是否需要继续留在该路线

**支援优先级**:
1. 小龙坑附近的战斗
2. 己方ADC被攻击
3. 己方防御塔被攻击
4. 小规模团战（2-3人）
5. 单带队友被抓

#### 3.5.3 野区控制

**打野路线**:
```
标准打野路线（红开）:
1. 红BUFF -> 三狼 -> 四鬼 -> 蓝BUFF -> 蛤蟆 -> 河蟹
2. 根据线上情况决定gank路线

标准打野路线（蓝开）:
1. 蓝BUFF -> 三狼 -> 四鬼 -> 红BUFF -> 蛤蟆 -> 河蟹
2. 根据线上情况决定gank路线
```

**野区入侵**:
- 当敌方打野在上半区时入侵下半区
- 队友在线上有优势时入侵
- 控制关键野区视野（使用扫描和真眼）

**野区防守**:
- 监控己方野区刷新时间
- 当敌方打野靠近时通知队友
- 在关键野区入口布置视野

#### 3.5.4 目标推进

**推进优先级**:
```
1. 小龙: 提供团队增益 [权重: 100]
   - 每6分钟刷新
   - 击杀后提供元素龙BUFF

2. 防御塔: 获得地图控制权 [权重: 90]
   - 提供视野和推进空间
   - 摧毁后获得金币奖励

3. 大龙: 提供强力BUFF [权重: 80]
   - 每7分钟刷新
   - 击杀后提供大龙BUFF

4. 水晶: 推进游戏胜利 [权重: 70]
   - 摧毁水晶后召唤超级兵

5. 基地: 游戏胜利 [权重: 200]
```

**推进时机**:
- 己方人数优势（至少多1人）
- 敌方关键技能在冷却
- 拥有大龙BUFF或小龙增益
- 兵线位置良好（有超级兵或大量小兵）
- 敌方防御塔血量较低

**推进策略**:
- 先清理敌方小兵
- 再攻击防御塔
- 注意敌方英雄防守
- 必要时越塔强杀

#### 3.5.5 团队协作

**协作策略**:
- 根据角色定位分工（坦克在前排，ADC在后排）
- 团战中各司其职（辅助保护ADC，刺客切后排）
- 资源分配优先级（小龙>防御塔>大龙）
- 支援响应机制（收到支援请求后5秒内响应）

**团战站位**:
- 坦克: 前排吸收伤害，保护后排
- 战士: 半肉半输出，切入敌方后排
- 刺客: 切入敌方后排，击杀ADC/APC
- ADC: 后排输出，保持安全位置
- 辅助: 保护ADC，提供控制和治疗

## 四、角色定位行为差异

### 4.1 上单(TOP)

**核心职责**:
- 对线压制
- 单带推进
- 团战前排

**行为特征**:
- 优先发育，保证血量（血量<40%时回城）
- 利用草丛进行换血（换血后退回草丛）
- 注意防gank（保持河道视野）
- 后期单带牵制（带线时注意小地图）
- 团战中顶在最前面，吸收伤害

**装备购买顺序**:
1. 防御装备（日炎、反甲、蓝盾）
2. 输出装备（黑切、九头蛇）
3. 鞋子（护甲鞋或魔抗鞋）

**技能加点**:
- 主Q或主W（根据英雄）
- 有R点R

### 4.2 打野(JUNGLE)

**核心职责**:
- 野区控制
- 线上gank
- 小龙/大龙控制

**行为特征**:
- 按固定路线清野（保持高效刷野）
- 观察线上机会进行gank（线上敌人压线时）
- 控制视野（在河道和三角草丛插眼）
- 争夺地图资源（优先小龙，其次大龙）
- 反蹲（预测敌方打野gank路线）

**gank时机**:
- 线上敌人压线过深
- 线上队友有控制技能
- 敌方打野在上半区
- 己方线上英雄血量健康

**刷野路线**:
- 红开: 红BUFF -> 三狼 -> 四鬼 -> 蓝BUFF -> 蛤蟆 -> 河蟹
- 蓝开: 蓝BUFF -> 三狼 -> 四鬼 -> 红BUFF -> 蛤蟆 -> 河蟹

### 4.3 中单(MID)

**核心职责**:
- 快速发育
- 支援上下路
- 团战输出

**行为特征**:
- 快速清线后支援（清线时间<20秒）
- 注意河道视野（防止被gank）
- 控制中路节奏（推线后游走）
- 在团战中寻找输出位置（避免被切）
- 优先击杀敌方后排

**支援路线**:
- 6级前: 主要支援打野（帮助入侵野区）
- 6级后: 优先支援小龙坑附近的战斗
- 后期: 跟随团队推进

**装备购买顺序**:
1. 核心输出装备（帽子、法穿鞋）
2. 防御装备（沙漏、女妖）
3. 法穿装备（虚空之杖）

### 4.4 ADC

**核心职责**:
- 持续输出
- 稳定发育
- 团战存活

**行为特征**:
- 专注补刀，保证经济（补刀成功率>80%）
- 团战中保持安全位置（距离敌方刺客>500）
- 利用走A最大化输出（攻击间隔<1秒）
- 注意防切（观察敌方刺客位置）
- 优先攻击最靠近自己的敌人

**对线策略**:
- 前期: 专注补刀，避免换血
- 中期: 配合辅助进行压制
- 后期: 团队核心输出

**装备购买顺序**:
1. 核心输出装备（无尽、饮血、电刀）
2. 防御装备（水银弯刀、复活甲）
3. 鞋子（攻速鞋）

### 4.5 辅助(SUPPORT)

**核心职责**:
- 保护ADC
- 视野控制
- 开团/控制

**行为特征**:
- 保护ADC，牺牲自己（ADC血量<50%时挡在前面）
- 提供视野（每分钟插1-2个眼）
- 先手开团或反手保护（根据英雄定位）
- 控制小龙/大龙视野（小龙刷新前布置视野）
- 购买辅助装备（眼石、香炉）

**辅助类型**:
- **坦克辅助**（日女、牛头）: 先手开团，保护ADC
- **保护辅助**（索拉卡、风女）: 治疗队友，提供护盾
- **开团辅助**（锤石、机器人）: 钩子技能开团
- **AP辅助**（莫甘娜、火男）: 提供控制和伤害

**装备购买顺序**:
1. 辅助装备（眼石、香炉、救赎）
2. 防御装备（坩埚、鸟盾）
3. 鞋子（五速鞋或CD鞋）

## 五、状态机设计

### 5.1 核心状态

```
IDLE (空闲)
    ↓
LANING (对线) ←→ COMBAT (战斗)
    ↓              ↓
RECALLED (回城)   CHASE (追击)
    ↓              ↓
SHOPPING (购物)   FLEEING (逃跑)
    ↓              ↓
RETURNING (返回线上)
    ↓
LANING (对线)
```

### 5.2 状态转换条件

| 当前状态 | 转换条件 | 目标状态 |
|---------|---------|---------|
| IDLE | 到达线上 | LANING |
| LANING | 发现敌人进入攻击范围 | COMBAT |
| LANING | 血量 < 30% 或蓝量 < 20% | RECALLED |
| LANING | 金币足够买核心装备 | RECALLED |
| LANING | 队友请求支援且支援评分 > 50 | SUPPORTING |
| COMBAT | 敌人逃跑且追击评分 > 50 | CHASE |
| COMBAT | 自身血量 < 20% 或逃跑评分 > 100 | FLEEING |
| COMBAT | 敌人死亡或超出范围 | LANING |
| CHASE | 目标超出追击范围（>1500） | LANING |
| CHASE | 目标死亡 | LANING |
| CHASE | 自身血量 < 30% | FLEEING |
| FLEEING | 到达安全区域（防御塔附近） | LANING |
| FLEEING | 血量恢复 > 50% | LANING |
| RECALLED | 到达基地 | SHOPPING |
| SHOPPING | 购买完成 | RETURNING |
| RETURNING | 到达线上 | LANING |
| SUPPORTING | 支援完成或战斗结束 | LANING |

### 7.1 Bot间通信

**通信内容**:
- 敌方位置信息（实时共享）

**通信方式**:
- 共享全局状态（所有Bot可见）
- 事件广播（当发生重要事件时）
- 目标标记系统（团战中标记优先目标）

**通信协议**:
```
消息格式:
{
    "sender": "BotID",
    "type": "MESSAGE_TYPE",
    "data": { ... },
    "timestamp": "游戏时间"
}

消息类型:
- ENEMY_SPOTTED: 发现敌人
- SUPPORT_REQUEST: 请求支援
- SKILL_READY: 技能就绪
- OBJECTIVE_THREAT: 目标受到威胁
- GANK_ALERT: gank警报
```

### 7.2 团队协作

**协作策略**:
- 根据角色定位分工（坦克在前排，ADC在后排）
- 团战中各司其职（辅助保护ADC，刺客切后排）
- 资源分配优先级（小龙>防御塔>大龙）
- 支援响应机制（收到支援请求后5秒内响应）

**团战协作**:
```
团战流程:
1. 辅助先手开团（控制技能）
2. 坦克跟进（吸收伤害）
3. 刺客切入（击杀敌方后排）
4. ADC输出（持续输出敌方前排）
5. 中单输出（技能清场）

团战目标优先级:
1. 敌方ADC（最高优先级）
2. 敌方APC（次高优先级）
3. 敌方刺客（第三优先级）
4. 敌方辅助（第四优先级）
5. 敌方坦克（最低优先级）
```

**资源协作**:
- 小龙: 辅助提供视野，打野控制，队友配合击杀
- 大龙: 全队配合，先清视野，再击杀
- 防御塔: ADC输出，坦克抗塔，辅助保护

### 10.2 核心算法

**目标选择算法**:
```csharp
AttackableUnit SelectTarget(List<AttackableUnit> enemies) {
    float bestScore = float.MinValue;
    AttackableUnit bestTarget = null;
    
    foreach (var enemy in enemies) {
        float score = EvaluateTarget(enemy);
        if (score > bestScore) {
            bestScore = score;
            bestTarget = enemy;
        }
    }
    
    return bestTarget;
}

float EvaluateTarget(AttackableUnit target) {
    float score = 0;
    
    // 正在攻击自己的敌人优先级最高
    if (target is Champion enemyChamp && enemyChamp.TargetUnit == champion) {
        score += 500;
    }
    
    // 血量越低优先级越高
    float hpPercent = target.Stats.CurrentHealth / target.Stats.HealthPoints.Total;
    score += (1 - hpPercent) * 300;
    
    // 距离越近优先级越高
    float distance = Vector2.Distance(champion.Position, target.Position);
    score += Math.Max(0, 200 - distance / 10);
    
    // ADC/APC优先级更高
    if (IsHighPriorityTarget(target)) {
        score += 200;
    }
    
    // 近战英雄更容易击杀
    if (target is Champion && ((Champion)target).IsMelee) {
        score += 50;
    }
    
    return score;
}
```

**威胁评估算法**:
```csharp
float EvaluateThreat(List<AttackableUnit> enemies) {
    float selfThreat = CalculateSelfThreat();
    float enemyThreat = CalculateEnemyThreat(enemies);
    
    return selfThreat - enemyThreat;
}

float CalculateSelfThreat() {
    float threat = 0;
    
    // 血量百分比
    threat += (champion.Stats.CurrentHealth / champion.Stats.HealthPoints.Total) * 100;
    
    // 可用技能数量
    int readySpells = champion.Spells.Count(s => s.CurrentCooldown <= 0);
    threat += readySpells * 50;
    
    // 装备评分
    float itemValue = champion.Inventory.Items.Sum(i => i.ItemData.Price);
    threat += itemValue / 100;
    
    return threat;
}

float CalculateEnemyThreat(List<AttackableUnit> enemies) {
    float threat = 0;
    
    // 敌方人数
    threat += enemies.Count * 80;
    
    // 敌方平均血量
    float avgHpPercent = enemies.Average(e => e.Stats.CurrentHealth / e.Stats.HealthPoints.Total);
    threat += avgHpPercent * 50;
    
    // 敌方可用技能数量
    int enemyReadySpells = enemies.OfType<Champion>()
        .Sum(c => c.Spells.Count(s => s.CurrentCooldown <= 0));
    threat += enemyReadySpells * 30;
    
    return threat;
}
```

## 十一、总结

LOL 4.20 AI Bot采用分层行为树架构，实现了从紧急反应到宏观决策的完整行为系统。核心设计亮点包括：

1. **动态威胁评估**：基于血量、技能状态、装备等多维度评估战斗风险，使Bot行为更加智能
2. **类人化操作**：实现走A、技能躲避、补刀等人类玩家常见操作
3. **角色差异化**：不同定位的英雄表现出截然不同的行为模式
4. **团队协作**：Bot之间能够进行有效的通信和配合
5. **可配置难度**：支持多种难度级别，满足不同玩家需求

通过模块化设计，每个行为模块可以独立开发和测试，同时支持不同角色定位的差异化行为。配合状态机和时间管理系统，Bot能够表现出符合游戏策略的智能行为，为玩家提供有挑战性的游戏体验。
